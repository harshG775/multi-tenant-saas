# AGENTS.md

## Stack

- TypeScript, React 19 (React Compiler enabled via Babel plugin)
- TanStack Start (SSR/server functions) + TanStack Router (file-based) + TanStack Query
- Vite 8 + Nitro server adapter
- Drizzle ORM `1.0.0-rc.4` + `@neondatabase/serverless` (Neon Postgres, HTTP driver)
- better-auth
- Tailwind CSS v4 + shadcn/ui
- **pnpm** (`packageManager: pnpm@11.1.2` in package.json) — not npm/yarn
- Node.js — no version pinned in package.json/`engines`

## Commands

```bash
pnpm install
pnpm dev              # dev server, http://localhost:3000
pnpm build             # production build -> dist/
pnpm preview            # preview production build
pnpm test               # vitest run (all tests)
pnpm vitest run path/to/file.test.ts   # single file — no dedicated script exists
pnpm lint                # eslint
pnpm format              # prettier --write . && eslint --fix
pnpm check                # prettier --check . (no writes)
pnpm db:generate           # generate migration from schema changes
pnpm db:migrate             # apply migrations
pnpm db:push                  # push schema directly, no migration file
pnpm db:studio                 # open Drizzle Studio
```

No `typecheck` script is defined. `tsconfig.json` has `noEmit: true`; run `pnpm exec tsc --noEmit` directly if you need a standalone check.

## Conventions

- Import alias: use `#/*`, not `@/*` (both resolve to `src/*`; `@/*` only exists for shadcn-generated files).
  `import { getTenantFn } from "#/modules/tenant"`
- New features live in `src/modules/<feature>/` (middleware/server-fn/types) with an `index.ts` barrel. Import only from the barrel, never a module's internal files.
- `src/lib/` is for generic, cross-feature utilities only (e.g. `cn()`, `normalizeHostname.ts`, the db/auth clients) — not feature-specific logic.
- Never hand-edit `src/routeTree.gen.ts` — it's generated by the TanStack Router Vite plugin on `dev`/`build`.
- Schema changes: edit `src/lib/db/schema/*.ts` (and `src/lib/db/relations/index.ts` if FKs changed) → `pnpm db:generate` → `pnpm db:migrate`. Never hand-write files under `src/lib/db/migrations/`.
- Env vars: add to `src/env.ts` in both the `server`/`client` schema block *and* `runtimeEnv` — client vars must be prefixed `VITE_`.
- Add UI components with `pnpm dlx shadcn@latest add <component>` — don't hand-write files under `src/components/ui/`.
- ESLint intentionally disables `import/no-cycle`, `import/order`, `sort-imports`, `@typescript-eslint/array-type`, `@typescript-eslint/require-await`, `pnpm/json-enforce-catalog` — don't manually reorder/sort imports to "fix" lint.
- In `src/routes/api/auth/$.ts`, requests are rebuilt from `request.url` + explicit headers, never `new Request(existingRequest, { headers })` — the clone form throws in this stack. Keep this pattern if touching that file.

## Boundaries

**Always**: run `pnpm lint` and `pnpm check` before finishing a change; regenerate migrations with `pnpm db:generate` instead of writing SQL by hand.

**Ask first**: running `pnpm db:migrate`/`pnpm db:push` against a real database; adding/upgrading dependencies; changing `trustedOrigins` or anything in `src/lib/auth/`; changing the `tenantMiddleware` lookup logic.

**Never**: commit `.env`/`.env.local`; hardcode `DATABASE_URL`, `BETTER_AUTH_SECRET`, or any API key; use `npm`/`yarn`; bypass `tenantMiddleware`/`context.tenant` for data access.
